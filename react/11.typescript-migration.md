## 1. 前言：拼错字段名是运行时才发现

第十篇典型卡点：

- `citation.snippet` 写成 `citation.excerpt`，界面空白，控制台 `undefined`。
- 不确定 `.jsx` 改 `.tsx` 后 Vite 要不要重配。
- 看见 `useState<Message[]>([])` 不懂尖括号在干什么。
- 怕「一次全改 TS」导致项目起不来。

**TypeScript**（TS，类型脚本）：JavaScript 的超集，增加**静态类型**；编译成普通 JS 给浏览器跑。  
通俗说：写代码时 IDE 就帮你查「这个对象有没有这个字段」，很多错在保存文件时就标红。

**静态类型**：在**运行前**（编辑、编译阶段）检查类型是否匹配。  
通俗说：做菜前对照菜谱清点食材，不是上桌才发现没放盐。

读完本文，你应该能做到：

1. 在现有 Vite React 项目启用 TS（或新建 `react-ts` 模板对照迁移）。
2. 定义 `Message`、`Citation`、`KbDocument` 等 `interface` 并在组件中使用。
3. 给 `ChatMessage`、`CitationList` 的 props 写类型。
4. 写 `fetchJSON<T>`，让 `GET /api/documents` 返回 `KbDocument[]`。
5. 按「types → utils → 叶子组件 → 页面」顺序渐进改 `.tsx`。

**前置阅读**：[（七）～（十）](07.sse-streaming-chat.md) 的聊天 / 引用 / 上传代码；[（一）ES6+](01.javascript-es6-quickstart.md) 对象与解构。

**环境**：Node 18+；延续 `frontend/` 项目。

### 1.1 本文边界

不深究：`extends` 复杂泛型、`satisfies`、声明合并、TS 5.x 全部编译选项。

目标：**核心 RAG 类型齐全，主要页面与组件改为 `.tsx` 且 `npm run build` 通过。**

## 2. TypeScript 是什么：带类型的 JavaScript

| | JavaScript | TypeScript |
|---|------------|------------|
| 扩展名 | `.js` / `.jsx` | `.ts` / `.tsx`（含 JSX） |
| 类型 | 运行时才暴露 | 编辑期检查 |
| 浏览器 | 直接跑（或打包后跑） | 先 `tsc` 或 Vite 转译成 JS |

**interface**（接口）：描述对象应有哪些字段及类型。  
通俗说：表格模板——`Citation` 必须有 `id`、`title`、`snippet`。

```typescript
interface Citation {
  id: number;
  title: string;
  snippet: string;
  source?: string;   // ? 表示可选
  page?: number | null;
}
```

**type**（类型别名）：给类型起名字，可表示联合等。  
通俗说：和 interface 类似，本篇用 `type Role = 'user' | 'assistant'` 很方便。

```typescript
type Role = 'user' | 'assistant';
```

Vite 开发时**不做完整类型检查**，只转译；`npm run build` 时会检查。建议加脚本 `"typecheck": "tsc --noEmit"`（§3）。

## 3. 在 Vite 里启用 TypeScript

两种路径：

| 路径 | 适合 |
|------|------|
| **A. 现有 JS 项目渐进** | 你已有第七～十篇 `frontend/` |
| **B. 新建 react-ts 对照** | 想从零干净起步 |

### 3.1 路径 A：渐进迁移

演示什么：安装 TS，增加配置文件，先把一个文件改成 `.tsx`。  
在 `frontend/` 执行：

```bash
npm install -D typescript @types/react @types/react-dom
```

新建 `tsconfig.json`（Vite 官方最小集）：

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",
    "strict": true
  },
  "include": ["src"]
}
```

`package.json` 增加：

```json
"typecheck": "tsc --noEmit"
```

把 `src/main.jsx` 重命名为 `src/main.tsx`，`index.html` 里 script 指向 `/src/main.tsx`。

**允许 JS 与 TS 共存**：未改的文件可保持 `.jsx`，Vite 都支持。

### 3.2 路径 B：新建对照（了解即可）

```bash
npm create vite@latest my-rag-ts -- --template react-ts
```

把第七～十篇的 `src` 逐步复制进去并改后缀即可。

预期：`npm run dev` 正常；`npm run typecheck` 无报错（迁移完成后）。

## 4. 先定义 RAG 核心类型

演示什么：集中放类型，避免每个文件各写一份。  
新建 `src/types/rag.ts`：

```typescript
export type Role = 'user' | 'assistant';

export type JobStatus = 'pending' | 'running' | 'done' | 'failed';

export interface Citation {
  id: number;
  title: string;
  snippet: string;
  source?: string;
  page?: number | null;
  score?: number;
}

export interface Message {
  id: string;
  role: Role;
  content: string;
  citations?: Citation[];
}

export interface KbDocument {
  id: string;
  filename: string;
  status: JobStatus;
  progress: number;
  error?: string | null;
  created_at: string;
}

/** SSE 流式事件：正文 token */
export interface SseTokenPayload {
  token: string;
}

/** SSE 流式事件：引用列表 */
export interface SseCitationsPayload {
  citations: Citation[];
}
```

**为何单独文件**：聊天、上传、调试台（第十三篇）共用同一套形状，改一处全局生效。

与第九篇字段对齐：`Citation.id` 对应脚注 `[1]`；`KbDocument` 对应第十篇列表行。

## 5. 组件 props：从 JSDoc 到 interface

演示什么：把 `ChatMessage.jsx` 改为 `ChatMessage.tsx`，与 [第九篇 §7](09.citation-source-ui.md) 行为一致，仅补上类型。

```tsx
import type { Citation, Role } from '../types/rag';
import { MarkdownBubble } from './MarkdownBubble';
import { CitationList } from './CitationList';

export interface ChatMessageProps {
  role: Role;
  content: string;
  citations?: Citation[];
  streaming?: boolean;
  onCitationSelect?: (c: Citation) => void;
}

interface AssistantContentProps {
  content: string;
  streaming: boolean;
}

function AssistantContent({ content, streaming }: AssistantContentProps) {
  if (!content) return <span style={{ color: '#6b7280' }}>…</span>;
  if (streaming) {
    return <div style={{ whiteSpace: 'pre-wrap' }}>{content}</div>;
  }
  return <MarkdownBubble content={content} />;
}

export function ChatMessage({
  role,
  content,
  citations,
  streaming = false,
  onCitationSelect,
}: ChatMessageProps) {
  const isUser = role === 'user';
  return (
    <div
      style={{
        display: 'flex',
        justifyContent: isUser ? 'flex-end' : 'flex-start',
        marginBottom: 12,
      }}
    >
      <div style={{ maxWidth: '80%' }}>
        <div
          style={{
            padding: '8px 12px',
            borderRadius: 12,
            background: isUser ? '#2563eb' : '#f3f4f6',
            color: isUser ? '#fff' : '#111',
          }}
        >
          {isUser ? (
            <div style={{ whiteSpace: 'pre-wrap' }}>{content}</div>
          ) : (
            <AssistantContent content={content} streaming={streaming} />
          )}
        </div>
        {!isUser && onCitationSelect && (
          <CitationList citations={citations} onSelect={onCitationSelect} />
        )}
      </div>
    </div>
  );
}
```

预期：与第九篇相同——流式时 `streaming={true}` 走纯文本；结束后走 `MarkdownBubble`；`citations` 到达后显示可点卡片。

`CitationList.tsx` 完整版（第九篇逻辑 + 类型）：

```tsx
import type { Citation } from '../types/rag';

export interface CitationListProps {
  citations?: Citation[];
  onSelect: (c: Citation) => void;
}

export function CitationList({ citations, onSelect }: CitationListProps) {
  if (!citations?.length) return null;

  return (
    <div style={{ marginTop: 8, display: 'flex', flexDirection: 'column', gap: 6 }}>
      <div style={{ fontSize: 12, color: '#6b7280' }}>参考来源</div>
      {citations.map((c) => (
        <button
          key={c.id}
          type="button"
          onClick={() => onSelect(c)}
          style={{
            textAlign: 'left',
            padding: '8px 10px',
            borderRadius: 8,
            border: '1px solid #e5e7eb',
            background: '#fff',
            cursor: 'pointer',
            fontSize: 13,
          }}
        >
          <span style={{ fontWeight: 600, marginRight: 6 }}>[{c.id}]</span>
          <span>{c.title}</span>
          {c.page != null && (
            <span style={{ color: '#6b7280', marginLeft: 6 }}>p.{c.page}</span>
          )}
        </button>
      ))}
    </div>
  );
}
```

**`import type`**：只导入类型，编译后擦除，不增加运行时代码。  
通俗说：跟类型有关的 `import` 标清楚，打包更干净。

先错后对：

```tsx
// ❌ role 拼错不会报错（JS）
function ChatMessage({ role }) {
  const isUser = role === 'usr';
}

// ✅ 联合类型约束
type Role = 'user' | 'assistant';
// role === 'usr' 会标红
```

## 6. fetch 与 JSON：泛型 fetchJSON

演示什么：让 TypeScript 知道 `res.json()` 之后是什么。  
`src/utils/fetchJSON.ts`：

```typescript
export async function fetchJSON<T>(url: string, init?: RequestInit): Promise<T> {
  const res = await fetch(url, init);
  if (!res.ok) {
    const text = await res.text().catch(() => '');
    throw new Error(text || `HTTP ${res.status}`);
  }
  return res.json() as Promise<T>;
}
```

使用（第十篇文档列表）：

```typescript
import type { KbDocument } from '../types/rag';
import { fetchJSON } from '../utils/fetchJSON';

const list = await fetchJSON<KbDocument[]>('/api/documents');
// list[0].filename ✅  list[0].fileName ❌ 编译器报错
```

**泛型 `<T>`**：调用时填入「期望的返回类型」占位符。  
通俗说：`fetchJSON` 是万能快递盒，`<KbDocument[]>` 告诉 TS 盒子里是文档数组。

`readSSEStream` 可加回调类型：

```typescript
export async function readSSEStream(
  res: Response,
  onToken: (token: string) => void,
  onCitations?: (citations: Citation[]) => void,
): Promise<void> {
  // … 实现同第九篇 …
}
```

## 7. useState 与事件：常见写法

```tsx
const [messages, setMessages] = useState<Message[]>([]);
const [activeCitation, setActiveCitation] = useState<Citation | null>(null);
const [input, setInput] = useState('');
```

**`useState<Citation | null>(null)`**：状态要么是 `Citation` 对象，要么是 `null`（侧栏未选中）。

表单事件：

```tsx
function onFileChange(e: React.ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0] ?? null;
  setFile(file);
}
```

`React.ChangeEvent<HTMLInputElement>` 是 React 提供的类型，不必背——IDE 补全即可。

**可选阅读**：`React.MouseEvent<HTMLButtonElement>` 用于 `onClick`；初学用 `Parameters` 或 `any` 过渡不推荐，尽量用上面几种。

## 8. 迁移顺序：先 utils 再组件

建议改文件顺序（每改一个跑 `npm run dev`）：

```text
1. src/types/rag.ts           （新建）
2. src/utils/fetchJSON.ts
3. src/utils/readSSEStream.ts
4. src/components/MarkdownBubble.tsx
5. src/components/CitationList.tsx
6. src/components/SourcePanel.tsx
7. src/components/ChatMessage.tsx
8. src/pages/ChatPage.tsx
9. src/pages/DocumentsPage.tsx
10. src/main.tsx
```

**叶子组件优先**：`ChatPage` 依赖下面组件，先保证子组件类型正确，页面错误会自然减少。

遇到第三方库无类型：

```bash
npm install -D @types/某个库   # 若有
```

`react-markdown` 等现代包常自带类型，一般不用 `@types`。

`npm run typecheck` 通过即可视为迁移里程碑；不必一天内改光所有 `.jsx`。

### 8.1 改前 / 改后：`DocumentsPage` 拉列表

演示什么：同一页面，从第十篇 JS + 手写 `fetch` 迁到 TS + `fetchJSON<KbDocument[]>`（第十二篇会再换成 `useQuery`）。

**改前**（第十篇 `DocumentsPage.jsx` 核心，节选）：

```jsx
const [documents, setDocuments] = useState([]);

useEffect(() => {
  async function load() {
    const res = await fetch('/api/documents');
    const list = await res.json();
    setDocuments(list);
    // … 轮询逻辑 …
  }
  load();
}, [pollVersion]);
```

**改后**（第十一篇过渡版，尚未上 Query）：

```tsx
import { useEffect, useState } from 'react';
import type { KbDocument } from '../types/rag';
import { fetchJSON } from '../utils/fetchJSON';

export function DocumentsPage() {
  const [documents, setDocuments] = useState<KbDocument[]>([]);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    let cancelled = false;

    async function load() {
      try {
        const list = await fetchJSON<KbDocument[]>('/api/documents');
        if (!cancelled) setDocuments(list);
      } catch (e) {
        if (!cancelled) setError((e as Error).message);
      }
    }

    load();
    return () => {
      cancelled = true;
    };
  }, []);

  if (error) return <p style={{ color: '#b91c1c' }}>{error}</p>;

  return (
    <div>
      {/* UploadForm、DocumentList 与第十篇相同；documents 已有类型 */}
      <DocumentList documents={documents} />
    </div>
  );
}
```

`list[0].filename` 拼成 `fileName` 时编辑器会标红——这就是本篇要的核心收益。轮询仍可先保留第十篇写法；[第十二篇](12.tanstack-query.md) 再用 `useDocumentsQuery` 删掉整段 `useEffect`。

## 9. 常见陷阱与 FAQ

### 9.1 陷阱一：到处写 `as any`

```typescript
// ❌ 失去 TS 意义
const doc = (await res.json()) as any;

// ✅
const doc = await fetchJSON<KbDocument>('/api/documents/1');
```

### 9.2 陷阱二：可选链与 strictNullChecks

`strict: true` 时，`citations` 可能是 `undefined`：

```tsx
{citations?.map((c) => ...)}  // ✅
{citations.map(...)}           // ❌ 可能 undefined
```

### 9.3 陷阱三：interface 与运行时校验

TS **只在编译期**存在；后端若返回怪 JSON，仍要在运行时 `res.ok` + 形状检查。真严谨可配合 **Zod**（进阶）。

### 9.4 FAQ

**Q：必须全部 .tsx 吗？**  
A：否，JS/TS 可混用；核心 RAG 路径改完即可。

**Q：和 Python Pydantic 关系？**  
A：后端 Pydantic 管 API；前端 `interface` 与之**手动对齐**（或 OpenAPI  codegen，进阶）。

**Q：`enum` 还是 union？**  
A：本篇 `JobStatus` 用字符串联合类型即可，简单够用。
