## 1. 前言：没有上传就没有 RAG

第九篇典型延伸问题：

- 引用里的 `docs/rag-intro.md` **谁上传的？**
- 第五篇 `POST` 发的是 `JSON.stringify`——**传 PDF 还能这样吗？**
- 上传后要等索引几分钟，页面怎么显示 **「处理中」** 而不是假死？
- 轮询 `setInterval` 和 `useEffect` 怎么配合、离开页面要不要停？

**Ingest**（入库 / 摄取）：把用户上传的原始文件解析、分块、向量化并写入索引的全过程。  
通俗说：从「一个 PDF」变成「能检索的一堆 chunk」。

**FormData**（表单数据对象）：浏览器内置 API，用 `multipart/form-data` 格式上传**文件字段 + 普通字段**。  
通俗说：寄快递时的「包裹 + 面单」，和只寄一张 JSON 纸条不同。

读完本文，你应该能做到：

1. 说明文件上传为何不能用 `Content-Type: application/json` 塞二进制。
2. 用 `FormData` + `fetch` POST 文件，并处理上传中的 loading / error。
3. 实现 FastAPI `UploadFile` 接口与内存任务状态（`pending` → `running` → `done` / `failed`）。
4. 用 `useEffect` 轮询 `GET /api/documents/{id}`，组件卸载时清除定时器。
5. 做出文档列表 + 进度条，并与第九篇聊天页串联路由。

## 3. FormData 与 multipart：和 JSON POST 有何不同

第五篇创建用户：

```javascript
fetch('/api/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name, email }),
});
```

文件上传**不要**自己设 `Content-Type: application/json`，也**不要**对二进制 `JSON.stringify`。

演示什么：用 `FormData` 装文件字段。  
预期：浏览器自动设 `multipart/form-data` 及 boundary。

```javascript
const form = new FormData();
form.append('file', file); // file 来自 <input type="file" />

const res = await fetch('/api/documents', {
  method: 'POST',
  body: form,
  // 不要写 Content-Type —— 浏览器会自动带 boundary
});
```

先错后对：

```javascript
// ❌
headers: { 'Content-Type': 'application/json' },
body: JSON.stringify({ file }),

// ✅
body: formData,
// 无 Content-Type 头（或让浏览器生成 multipart）
```

| 对比 | JSON POST（第五篇） | 文件上传（本篇） |
|------|---------------------|------------------|
| body | 字符串 | `FormData` |
| Content-Type | `application/json` | `multipart/form-data`（自动） |
| 典型用途 | 创建用户、聊天 JSON | 上传 PDF、图片 |

## 4. 后端：上传接口 + 内存任务状态机

演示什么：接收文件、创建任务记录、后台协程模拟 ingest 进度。  
前置：`pip install fastapi uvicorn python-multipart`（`UploadFile` 需要 **python-multipart**）。

`requirements.txt` 增加：

```text
python-multipart>=0.0.6
```

`backend/main.py`（可与第七～九篇共存）：

```python
import asyncio
import uuid
from datetime import datetime, timezone
from enum import Enum

from fastapi import FastAPI, UploadFile, File, HTTPException, BackgroundTasks
from pydantic import BaseModel

app = FastAPI(title="KB Upload Demo")


class JobStatus(str, Enum):
    pending = "pending"
    running = "running"
    done = "done"
    failed = "failed"


# 内存任务表：生产环境换 PostgreSQL
_documents: dict[str, dict] = {}


class DocumentOut(BaseModel):
    id: str
    filename: str
    status: JobStatus
    progress: int  # 0-100
    error: str | None = None
    created_at: str


async def fake_ingest(document_id: str):
    doc = _documents[document_id]
    try:
        doc["status"] = JobStatus.running
        for p in range(0, 101, 10):
            doc["progress"] = p
            await asyncio.sleep(0.4)
        doc["status"] = JobStatus.done
        doc["progress"] = 100
    except Exception as e:
        doc["status"] = JobStatus.failed
        doc["error"] = str(e)


@app.post("/api/documents", response_model=DocumentOut, status_code=201)
async def upload_document(
    background_tasks: BackgroundTasks,
    file: UploadFile = File(...),
):
    if not file.filename:
        raise HTTPException(400, "缺少文件名")

    doc_id = str(uuid.uuid4())
    content = await file.read()  # 演示：读入内存；大文件应流式写磁盘
    _documents[doc_id] = {
        "id": doc_id,
        "filename": file.filename,
        "status": JobStatus.pending,
        "progress": 0,
        "error": None,
        "created_at": datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ"),
        "size": len(content),
    }
    background_tasks.add_task(fake_ingest, doc_id)
    return _documents[doc_id]


@app.get("/api/documents", response_model=list[DocumentOut])
def list_documents():
    return sorted(_documents.values(), key=lambda d: d["created_at"], reverse=True)


@app.get("/api/documents/{doc_id}", response_model=DocumentOut)
def get_document(doc_id: str):
    if doc_id not in _documents:
        raise HTTPException(404, "文档不存在")
    return _documents[doc_id]
```

**BackgroundTasks**（FastAPI 后台任务）：在响应返回给客户端**之后**执行函数。  
通俗说：先回执「已收件」，再在后台慢慢索引。

**状态机**：`pending` → `running` → `done` | `failed`。前端根据 `status` 显示徽章，根据 `progress` 画进度条。

自测：

```bash
curl -X POST http://localhost:8000/api/documents \
  -F "file=@./README.md"
```

预期：返回 `201` + `id`；多次 `GET /api/documents/{id}` 见 `progress` 递增。

## 5. 前端：选择文件与上传三态

演示什么：`input[type=file]`、上传按钮、三态 UI（与第五篇提交三态同构）。  
新建 `src/components/UploadForm.jsx`：

```jsx
import { useState } from 'react';

export function UploadForm({ onUploaded }) {
  const [file, setFile] = useState(null);
  const [uploading, setUploading] = useState(false);
  const [error, setError] = useState(null);

  async function handleSubmit(e) {
    e.preventDefault();
    if (!file || uploading) return;

    setUploading(true);
    setError(null);

    const form = new FormData();
    form.append('file', file);

    try {
      const res = await fetch('/api/documents', {
        method: 'POST',
        body: form,
      });
      if (!res.ok) {
        const text = await res.text().catch(() => '');
        throw new Error(text || `HTTP ${res.status}`);
      }
      const doc = await res.json();
      setFile(null);
      onUploaded?.(doc);
    } catch (err) {
      setError(err.message);
    } finally {
      setUploading(false);
    }
  }

  return (
    <form onSubmit={handleSubmit} style={{ marginBottom: 16 }}>
      <input
        type="file"
        accept=".md,.txt,.pdf"
        onChange={(e) => setFile(e.target.files?.[0] ?? null)}
        disabled={uploading}
      />
      <button type="submit" disabled={!file || uploading} style={{ marginLeft: 8 }}>
        {uploading ? '上传中…' : '上传并索引'}
      </button>
      {error && <p style={{ color: '#b91c1c' }}>{error}</p>}
    </form>
  );
}
```

**`accept`**：限制可选后缀；真正校验仍应在后端做白名单。

**`onUploaded(doc)`**：把新文档交给父组件加入列表并启动轮询（§6）。

## 6. 轮询任务状态：useEffect 与清理

索引在后台跑，前端需**周期性** `GET` 单文档或列表，直到 `done` / `failed`。

**轮询**（Polling）：隔固定秒数重复请求，直到满足停止条件。  
通俗说：每隔几秒问一次「好了吗？」——比 WebSocket 笨，但实现简单。

演示什么：自定义 hook `useDocumentPoll`。  
前置：第三篇 `useEffect` 清理函数。

`src/hooks/useDocumentPoll.js`：

```javascript
import { useEffect, useState } from 'react';

export function useDocumentPoll(documentId, intervalMs = 1500) {
  const [doc, setDoc] = useState(null);
  const [error, setError] = useState(null);

  useEffect(() => {
    if (!documentId) return;

    let cancelled = false;
    let timerId;

    async function tick() {
      try {
        const res = await fetch(`/api/documents/${documentId}`);
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const data = await res.json();
        if (cancelled) return;
        setDoc(data);

        if (data.status === 'done' || data.status === 'failed') {
          return; // 终态，不再调度
        }
        timerId = setTimeout(tick, intervalMs);
      } catch (err) {
        if (!cancelled) setError(err.message);
      }
    }

    tick();

    return () => {
      cancelled = true;
      clearTimeout(timerId);
    };
  }, [documentId, intervalMs]);

  return { doc, error };
}
```

**清理函数** `return () => { cancelled = true; clearTimeout(...) }`：组件卸载或 `documentId` 变化时**停止轮询**，避免内存泄漏与 setState 警告。  
通俗说：人走了就别再问「好了吗」。

先错后对：

```javascript
// ❌ setInterval 永不 clear，离开页面仍在请求
useEffect(() => {
  setInterval(() => fetch(...), 1500);
}, []);

// ✅ 终态停止 + unmount 清理
```

**副作用**：轮询期间会多次 `setDoc`，列表页应对**每个进行中的 id** 各开一个 poll，或只 poll 整个列表 `GET /api/documents`（文档少时更简单）。

列表页简化方案——轮询整表：

```javascript
useEffect(() => {
  let cancelled = false;
  let timerId;

  async function load() {
    const res = await fetch('/api/documents');
    const list = await res.json();
    if (cancelled) return;
    setDocuments(list);
    const hasRunning = list.some((d) => d.status === 'pending' || d.status === 'running');
    if (hasRunning) timerId = setTimeout(load, 1500);
  }

  load();
  return () => {
    cancelled = true;
    clearTimeout(timerId);
  };
}, []);
```

有任意任务未结束时继续轮询；全部 `done`/`failed` 后自动停。

**与 §8 实战的关系**：下面 `DocumentsPage` 采用**整表轮询**（文档少时最简单）。§6 的 `useDocumentPoll(documentId)` 适合「只盯一条上传任务」——二选一即可，不要两套同时开。若列表曾全部 `done`、轮询已停，**新上传后必须重启轮询**（§8 用 `pollVersion` 处理）；第十二篇 `useQuery` + `refetchInterval` 可自动规避此问题。

## 7. DocumentList：文档与进度 UI

演示什么：按 `status` 显示徽章，`running` 时显示进度条。  
新建 `src/components/DocumentList.jsx`：

```jsx
const STATUS_LABEL = {
  pending: '排队中',
  running: '索引中',
  done: '已完成',
  failed: '失败',
};

export function DocumentList({ documents }) {
  if (!documents.length) {
    return <p style={{ color: '#6b7280' }}>暂无文档，请先上传。</p>;
  }

  return (
    <ul style={{ listStyle: 'none', padding: 0, margin: 0 }}>
      {documents.map((d) => (
        <li
          key={d.id}
          style={{
            border: '1px solid #e5e7eb',
            borderRadius: 8,
            padding: 12,
            marginBottom: 8,
          }}
        >
          <div style={{ display: 'flex', justifyContent: 'space-between' }}>
            <strong>{d.filename}</strong>
            <span>{STATUS_LABEL[d.status] ?? d.status}</span>
          </div>
          {(d.status === 'pending' || d.status === 'running') && (
            <div style={{ marginTop: 8 }}>
              <div
                style={{
                  height: 6,
                  background: '#e5e7eb',
                  borderRadius: 3,
                  overflow: 'hidden',
                }}
              >
                <div
                  style={{
                    width: `${d.progress}%`,
                    height: '100%',
                    background: '#2563eb',
                    transition: 'width 0.3s',
                  }}
                />
              </div>
              <span style={{ fontSize: 12, color: '#6b7280' }}>{d.progress}%</span>
            </div>
          )}
          {d.status === 'failed' && (
            <p style={{ color: '#b91c1c', fontSize: 13, marginTop: 8 }}>{d.error}</p>
          )}
        </li>
      ))}
    </ul>
  );
}
```

`pending` 且 `progress === 0` 时仍显示空条——用户知道任务已创建。

```jsx
import { useEffect, useState } from 'react';
import { Link } from 'react-router-dom';
import { UploadForm } from '../components/UploadForm';
import { DocumentList } from '../components/DocumentList';

export function DocumentsPage() {
  const [documents, setDocuments] = useState([]);
  // 上传后若轮询已停，递增 pollVersion 以重启 §6 的 effect
  const [pollVersion, setPollVersion] = useState(0);

  useEffect(() => {
    let cancelled = false;
    let timerId;

    async function load() {
      try {
        const res = await fetch('/api/documents');
        const list = await res.json();
        if (cancelled) return;
        setDocuments(list);
        const active = list.some(
          (d) => d.status === 'pending' || d.status === 'running'
        );
        if (active) timerId = setTimeout(load, 1500);
      } catch {
        /* 可加 error state */
      }
    }

    load();
    return () => {
      cancelled = true;
      clearTimeout(timerId);
    };
  }, [pollVersion]);

  function handleUploaded(doc) {
    setDocuments((prev) => [doc, ...prev]);
    setPollVersion((v) => v + 1);
  }

  return (
    <div style={{ maxWidth: 640, margin: '0 auto', padding: 24 }}>
      <div style={{ display: 'flex', justifyContent: 'space-between', alignItems: 'center' }}>
        <h1>知识库文档</h1>
        <Link to="/chat">去问答 →</Link>
      </div>
      <UploadForm onUploaded={handleUploaded} />
      <DocumentList documents={documents} />
    </div>
  );
}
```

## 9. 常见陷阱与 FAQ

### 9.1 陷阱一：手动写错 Content-Type

`multipart` 需要 boundary；手写 `application/json` 会导致后端解析失败。

### 9.2 陷阱二：轮询不设终态

`done` 后仍每 1.5s 请求——浪费。见 §6 在终态 `return`。

### 9.3 陷阱三：忘记 python-multipart

FastAPI 报 `UploadFile` 相关 422/安装提示——`pip install python-multipart`。

### 9.4 陷阱四：上传与轮询共用一个 loading

上传结束即可放开按钮；索引进度由列表轮询展示，别整页锁死。

### 9.5 陷阱五：超大文件一次 read()

演示代码 `await file.read()` 仅适合小文件；生产应流式写磁盘 + 队列。

### 9.6 FAQ

**Q：能用 React Query 吗？**  
A：`useQuery({ refetchInterval: 1500 })` 等价；学会手写轮询后迁移很容易。

**Q：上传进度百分比？**  
A：需 `XMLHttpRequest` 或 `fetch` + `ReadableStream` 监听 upload events——本篇用 indeterminate「上传中」即可。

**Q：多文件？**  
A：`form.append('files', f)` 多次 + 后端 `List[UploadFile]`——扩展作业。
