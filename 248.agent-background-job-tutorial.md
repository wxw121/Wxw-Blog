# AI Agent 工程（三十五）：Agent 后台任务

> 当 Agent 超过普通 HTTP 延迟预算，就应该返回 task_id，把执行交给后台 worker，并通过事件报告进度。

---

## 你会学到什么

- 设计异步任务 API。
- 使用队列、worker 和任务状态。
- 处理取消、重试和进度推送。
- 防止重复消费和并发推进。

## 它解决什么问题

长 Agent 同步运行会导致：

- 网关超时。
- 浏览器断开后任务状态未知。
- 用户重复提交。
- worker 重启丢失进度。
- 无法等待 Human Approval。

统一术语：**State、Transition、Checkpoint、Resume、Human Approval**。

## 最小示例

```python
from fastapi import FastAPI, status

app = FastAPI()


@app.post("/agent/tasks", status_code=status.HTTP_202_ACCEPTED)
async def create_task(request: CreateAgentTask) -> dict:
    task = task_service.create(request)
    queue.publish("agent.task.created", {"task_id": task.id})
    return {
        "task_id": task.id,
        "status": task.status,
        "status_url": f"/agent/tasks/{task.id}",
    }
```

查询：

```json
{
  "task_id": "TASK-901",
  "state": "running",
  "current_step": 3,
  "message": "正在验证引用"
}
```

## 工程化版本

### API

```text
POST /agent/tasks
GET /agent/tasks/{task_id}
POST /agent/tasks/{task_id}/cancel
GET /agent/tasks/{task_id}/events
POST /agent/tasks/{task_id}/approvals/{approval_id}
```

### Worker

worker 获取 lease，加载 Checkpoint，推进一个或有限步骤，然后保存并释放 lease。

```python
def process_task(task_id: str) -> None:
    with task_lease(task_id, ttl_seconds=30):
        state = checkpoint_store.load(task_id)
        state = orchestrator.advance(state)
        checkpoint_store.save(state)
        publish_state_event(state)
```

### 进度事件

只发布用户可理解摘要：

```json
{
  "event": "step.completed",
  "task_id": "TASK-901",
  "step": 3,
  "label": "引用验证完成"
}
```

不要通过 SSE 暴露完整工具参数或敏感 observation。

## 常见失败模式

- API 返回 200 但任务尚未完成。
- 队列重复投递导致两个 worker 执行。
- 前端断开就取消任务。
- cancel 只改状态，不通知正在运行的工具。
- Checkpoint 保存失败仍确认队列消息。
- 进度事件包含内部推理或敏感数据。

## 什么时候不要这么做

任务能在稳定低延迟内完成时同步接口更简单。

不要用队列掩盖没有停止条件的 Agent；后台无限循环仍然是故障。

## 生产环境注意事项

- 创建任务使用客户端幂等键。
- 队列按至少一次投递设计，consumer 幂等。
- lease 防止并发推进。
- State Transition 后再确认消息。
- cancel 与 Activity 协作。
- waiting_for_approval 不重复入队。

## 如何观测和评测

指标：

- 队列等待时间。
- worker 执行时间。
- 任务各 State 数量。
- 重复消费拦截。
- cancel 响应时间。
- Checkpoint / Resume 成功率。
- Human Approval 等待时间。

## 和 RAG / 后端 / 前端的关系

- RAG 长检索和批量分析由 worker 执行。
- 后端提供 task API 和事件。
- 前端用 SSE 或轮询展示进度。
- Human Approval 后发布 Resume 事件。

## 面试怎么讲

> 长 Agent 返回 202 和 task_id，通过队列驱动 worker。worker 获取 lease、加载 Checkpoint、推进 State、保存后发布事件。队列按至少一次投递设计，所以创建、消费和工具都要幂等。前端断线不等于取消，取消通过显式 API 传播。
