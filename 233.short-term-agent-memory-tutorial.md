# AI Agent 工程（二十）：短期记忆与会话上下文

> 短期记忆服务于当前会话：保留目标、约束和活跃实体，同时把冗长历史压缩到可控 token 预算。

---

## 你会学到什么

- 设计会话摘要和活跃实体。
- 使用滑动窗口与摘要组合。
- 防止摘要丢失约束或传播错误。
- 判断什么时候需要重新摘要。

## 它解决什么问题

长对话全部回传模型会导致：

- token 成本线性增长。
- 早期关键约束被挤出上下文。
- 工具原始结果重复出现。
- 历史中的恶意指令持续生效。

短期记忆只保留完成当前会话所需的信息。

## 最小示例

```python
from pydantic import BaseModel, Field


class SessionMemory(BaseModel):
    session_id: str
    current_goal: str
    constraints: list[str] = Field(default_factory=list)
    active_entities: dict[str, str] = Field(default_factory=dict)
    verified_facts: list[str] = Field(default_factory=list)
    summary: str = ""
    summarized_until_message_id: str | None = None
```

构造模型上下文：

```python
def build_context(
    memory: SessionMemory,
    recent_messages: list[Message],
    tool_observations: list[dict],
) -> list[dict]:
    return [
        {"role": "system", "content": SYSTEM_RULES},
        {"role": "system", "content": memory.model_dump_json()},
        *[message.to_model() for message in recent_messages[-8:]],
        *normalize_observations(tool_observations),
    ]
```

## 工程化版本

推荐“结构化状态 + 摘要 + 最近窗口”：

```text
结构化：goal、constraints、entities、verified_facts
摘要：较早对话的压缩说明
窗口：最近 6–10 条原始消息
```

写入判断：

| 信息类型 | 是否写入记忆 | 原因 | 删除策略 |
|---|---|---|---|
| 当前任务目标 | 写短期记忆 | 后续轮次必须保留 | 会话结束删除或归档 |
| 用户临时要求“这次用表格” | 写短期记忆 | 只对当前会话有效 | 会话结束删除 |
| 最近工具证据摘要 | 写短期记忆 | 支持下一步决策 | 任务结束删除 |
| 完整工具原始响应 | 否 | 占用大且可能敏感 | 放受控对象存储 |
| 用户长期语言偏好 | 不写短期，候选长期记忆 | 跨会话有价值 | 经确认后按长期策略 |

### 摘要必须保留不可丢字段

不要只让模型自由总结。输出结构化摘要：

```json
{
  "goal": "解释客户无法查看报表",
  "constraints": ["只使用内部资料", "不要修改权限"],
  "verified_facts": ["套餐支持高级报表"],
  "open_questions": ["用户角色是否有 report:read"]
}
```

### 摘要是派生数据

原消息是事实来源，摘要可以重建。发现摘要错误时，应回到原消息修复，而不是继续在错误摘要上追加。

## 常见失败模式

- 每轮重新总结全部历史。
- 摘要遗漏否定词和安全约束。
- 把未验证推测写入 verified_facts。
- 最近窗口太小，用户纠正信息被丢失。
- 不限制工具 observation 大小。
- 多个并发请求覆盖同一 session memory。

## 什么时候不要这么做

短对话不需要摘要，直接使用最近消息更准确。

不能只靠摘要保存合规审计；审计仍需原始事件。

关键交易参数应保存在任务状态和批准对象，不放自由文本摘要。

## 生产环境注意事项

- SessionMemory 带 version，使用乐观锁。
- 摘要器只接收当前用户可见数据。
- 会话过期时间明确。
- 用户删除会话时同步删除短期 Memory。
- Prompt Injection 内容不能被摘要成系统规则。
- 把系统指令和 Memory 放在不同结构中。

## 如何观测和评测

评测摘要：

- 目标保留率。
- 约束保留率。
- verified_facts 准确率。
- open_questions 完整率。
- 压缩比。
- 摘要后任务完成率变化。

维护包含纠正、否定、指代和多工具结果的对话集。

## 和 RAG / 后端 / 前端的关系

- RAG 证据只把必要摘要和引用写入短期 Memory。
- 后端负责版本、过期和并发更新。
- 前端可展示“本会话当前目标”，允许用户纠正。
- 长期偏好候选应显式征求用户同意。

## 面试怎么讲

> 短期记忆使用结构化目标、约束和已验证事实，加上旧消息摘要和最近滑动窗口。摘要是派生数据，不是审计真相；关键交易参数进入任务状态。摘要要保留否定和安全约束，并通过版本控制避免并发覆盖。
